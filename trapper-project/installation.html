Instalación

TRAPPER se proporciona como contenedor de Docker y puede ser desplegado en servidores basados en Linux y Windows. Sin embargo, recomendamos una distribución de Linux basada en Debian, por ejemplo Ubuntu Server. TRAPPER utiliza Docker Compose, una herramienta para definir y ejecutar aplicaciones de contenedores Docker multi-contenedor. Con esta tecnología, todos los servicios básicos necesarios para ejecutar TRAPPER (incluidos la base de datos y el servidor FTP) pueden ser desplegados y configurados automáticamente en un servidor dedicado con la posibilidad de configurar servicios específicos manualmente (por ejemplo, un escenario con un servidor de base de datos externo).

Instalación de Docker Community Edition (CE) & Docker Compose

  1. Siguiendo los pasos de la documentación oficial de Docker:

    • Ubuntu: https://docs.docker.com/install/linux/docker-ce/ubuntu/

    • Windows: https://hub.docker.com/editions/community/docker-ce-desktop-windows

  2. Instalar Docker Compose: https://docs.docker.com/compose/

Nota

Para aprender más sobre la tecnología de Docker, incluida la gestión de imágenes y contenedores de Docker, por favor lea la documentación oficial de Docker.

Obtener el código fuente de TRAPPER

  1. Primero, clone el código fuente de TRAPPER en tu repositorio local:

    $ git clone https://gitlab.com/trapper-project/trapper.git

  2. Para obtener la versión más actualizada de TRAPPER, cambie a la rama desarrollo:

    $ git checkout desarrollo

Nota

Es mucho más fácil utilizar Git Bash para clonar repositorios de Git en Windows en lugar de usar una línea de comandos estándar: https://git-scm.com/download/win

Ejecutar TRAPPER

Nota

Si necesita una configuración personalizada de TRAPPER (por ejemplo, base de datos externa, FTP y servicios de correo electrónico) por favor consulte la sección Configuración para obtener más detalles.

  1. Copie el archivo trapper.env al archivo .env, puede hacerlo utilizando el comando:

    $ cp trapper.env .env

  2. Ajuste las variables en el archivo .env si utiliza una instalación no estándar, por ejemplo, la base de datos PostgreSQL externa.

  3. Ejecute TRAPPER con el siguiente comando:

    • Producción

      $ ./start.sh -pb prod

    • Desarrollo

      $ ./start.sh -pb dev

Nota

Las banderas -p y -b indican al script que pulsa todas las imágenes de Docker requeridas y que construya la imagen trapper localmente. Para aprender más sobre todas las opciones de ejecución posibles simplemente escriba ./start.sh. Si necesita una configuración personalizada, por favor consulte la sección Configuración para obtener más detalles.

Certificados SSL

Si ya tiene certificados SSL para su sitio web de TRAPPER, puede utilizarlos proporcionando las rutas correctas en el archivo de configuración .env. Consulte la sección Certificados SSL para obtener más detalles. Estos certificados se utilizarán para configurar el servidor proxy nginx dentro de un contenedor dedicado. Si no se proporcionan certificados, se generarán automáticamente utilizando la herramienta openssl.

Nota

Puede obtener certificados SSL gratuitos de Let’s Encrypt.
